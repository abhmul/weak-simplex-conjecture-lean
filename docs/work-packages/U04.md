## Frozen public API

```lean
/-- The centered product theorem at singular covariance for continuous factors. -/
theorem centered_product_of_continuous
    {m : ℕ}
    (R : Matrix (Fin m) (Fin m) ℝ)
    (hR : IsCorrelation R)
    (f : Fin m → ℝ → ℝ)
    (hf_cont : ∀ i, Continuous (f i))
    (hf_meas : ∀ i, Measurable (f i))
    (hf_nonneg : ∀ i x, 0 ≤ f i x)
    (hf_bounded : ∀ i, Bornology.IsBounded (Set.range (f i)))
    (hf_lc : ∀ i, IsLogConcave (fun x ↦ ENNReal.ofReal (f i x)))
    (hf_mass_pos : ∀ i, 0 < ∫ x, f i x ∂gaussianReal 0 1)
    (hf_barycenter : ∀ i, ∫ x, x * f i x ∂gaussianReal 0 1 = 0) :
    (∏ i, ∫ x, f i x ∂gaussianReal 0 1) ≤
      ∫ x, ∏ i, f i (x i) ∂multivariateGaussian 0 R
```

## Method

Apply `centered_product_of_posDef` to `regularizedCovariance R (regularizationEpsilon n)`. Use U01 to supply positive definiteness, unit diagonal, and convergence in distribution. Prove that the finite coordinate product is continuous and bounded from the stated factor hypotheses, then pass its real integral to the limit. The left side is independent of the regularization parameter.

Prefer a packaged bounded-continuous integral theorem for convergence in distribution. Use the explicit U01 coupling and dominated convergence only if the exact Portmanteau interface is materially harder.

## Search and prototype record

Lean LSP searches for `TendstoInDistribution.integral`, `integral_continuous_bounded`, and `tendsto_of_tendstoInDistribution` returned no declaration. Source search located `ProbabilityMeasure.tendsto_iff_forall_integral_tendsto` in `Mathlib.MeasureTheory.Measure.ProbabilityMeasure`. A compile-checked `/tmp` prototype confirmed that applying this characterization to `TendstoInDistribution.tendsto` removes both `Measure.map id` terms by simplification. `BoundedContinuousFunction`'s pointwise commutative-ring structure supplies the finite product of coordinate factors.

## Implementation log

`ContinuousCenteredProduct.lean` packages each coordinate factor as a bounded continuous function, applies `centered_product_of_posDef` to every dyadic regularization, and closes the inequality under the Portmanteau limit with `ge_of_tendsto`. The file gate and full build pass, and the permanent axiom audit reports exactly `propext`, `Classical.choice`, and `Quot.sound` for the acceptance theorem.

## Forbidden shortcuts

- no theorem for arbitrary discontinuous factors;
- no classification of log-concave discontinuities;
- no change to `centered_product_of_posDef`;
- no stronger covariance or factor assumptions;
- no placeholders or project axioms.
