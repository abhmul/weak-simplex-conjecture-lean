## Frozen public API

The following names, types, and transparent definitions are frozen for this pass:

```lean
abbrev Coord (m : ℕ) := EuclideanSpace ℝ (Fin m)

def Coord.ofFun {m : ℕ} (x : Fin m → ℝ) : Coord m := WithLp.toLp 2 x
def Coord.toFun {m : ℕ} (x : Coord m) : Fin m → ℝ := WithLp.ofLp x

def coordinateMap {m : ℕ} (f : ℝ → ℝ) (x : Coord m) : Coord m :=
  Coord.ofFun (fun i ↦ f (x i))

def allOnesVector (m : ℕ) : Coord m := Coord.ofFun (fun _ ↦ 1)

def lowerOrthant {m : ℕ} (c : ℝ) : Set (Coord m) := {x | ∀ i, x i ≤ c}

def allOnesMatrix (m : ℕ) : Matrix (Fin m) (Fin m) ℝ := fun _ _ ↦ 1

def matrixMul {m : ℕ} (R : Matrix (Fin m) (Fin m) ℝ) (x : Coord m) : Coord m :=
  Matrix.toEuclideanCLM R x

def qform {m : ℕ} (R : Matrix (Fin m) (Fin m) ℝ) (x : Coord m) : ℝ :=
  ⟪x, matrixMul R x⟫_ℝ

def IsCorrelation {m : ℕ} (R : Matrix (Fin m) (Fin m) ℝ) : Prop :=
  R.PosSemidef ∧ ∀ i, R i i = 1

def IsWeakSimplexCov {m : ℕ} (R : Matrix (Fin m) (Fin m) ℝ) : Prop :=
  IsCorrelation R ∧
    (R - (1 / (m : ℝ)) • allOnesMatrix m).PosSemidef
```

Also provide the four `Coord.ofFun`/`Coord.toFun` evaluation and inverse simp lemmas, `matrixMul_ofFun`, and `qform_eq_dotProduct`. Small all-ones or finite sum/product lemmas may be added only when they express these wrappers or remove an observed downstream coercion.

The symbol `1` for matrices is the identity matrix, not `allOnesMatrix`; do not conflate them. Use the project `qform` wrapper rather than introducing `Matrix.toQuadraticForm'` downstream.

## Implementation log

- `WithLp.toLp_ofLp x` and `WithLp.ofLp_toLp x` treated `x` as the explicit exponent; the pinned API requires `WithLp.toLp_ofLp 2 x` and `WithLp.ofLp_toLp 2 x`.
- The coordinate-projection measurability goal required `(𝕜 := ℝ)` and `simpa only [EuclideanSpace.coe_proj]`; the direct unannotated projection did not unify with `fun x : Coord m ↦ x.ofLp i`.
- `Matrix.toEuclideanCLM R x` left the scalar equivalence ambiguous; the stable call is `Matrix.toEuclideanCLM (𝕜 := ℝ) R x`.
- Suffixed real inner-product notation requires `open scoped InnerProductSpace`; `RealInnerProductSpace` did not provide it.
- No finite sum/product bridge was added because the pinned `Fin.sum_const` and `Fin.prod_const` declarations suffice and no downstream coercion need was observed.
